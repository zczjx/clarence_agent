# 运维问题分诊流程

## 1. 澄清（先问清再跑命令）

- **现象**：错误信息、卡住还是崩溃、是否可复现
- **范围**：单个服务 / 整机 / 仅某项目
- **时间线**：何时开始、最近是否更新系统或依赖

## 2. 快速切片

- **对应服务的日志**：`journalctl -u <unit> -n 150 --no-pager` 或项目目录下的 `*.log`
- **资源是否打满**：`free -h`，`df -h`，`uptime`
- **端口是否监听**：`ss -tulpn | grep <port>`
- **进程是否存活**：`ps aux | grep -E '<name>|PID'`

## 3. 收敛

- 先定 **一层根因假设**（配置 / 资源 / 网络 / 依赖版本），再给 **下一步一条命令** 验证
- 避免一次抛出 20 条命令；每条说明**验证什么**

## 4. 收尾

- 总结：**根因** + **已做操作** + **如何验证恢复**
- 若需持久修复，说明是否要改文件/服务并让用户确认
